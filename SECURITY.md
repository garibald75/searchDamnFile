# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for a vulnerability that could put users at
risk.

If GitHub private vulnerability reporting is enabled for this repository, use it.
Otherwise, open a minimal public issue asking for a private maintainer contact
without including exploit details.

Useful details include:

- Affected version or commit.
- Steps to reproduce.
- Expected impact.
- Whether the issue requires a malicious file, folder name, path, or archive.

## Supported Versions

Security fixes are handled on the latest source version in the default branch.
